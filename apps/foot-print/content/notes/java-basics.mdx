## 자바란?

자바는 **표준 스펙**과 **구현**으로 나뉘어져 있다.

### 자바 표준 스펙

- 자바는 이렇게 만들어야 한다는 **설계도이자 문서**
- 이 표준 스펙을 기반으로 여러 회사에서 실제 작동하는 자바를 만든다
- 자바 표준 스펙은 **자바 커뮤니티 프로세스 JCP**를 통해 관리된다

### 다양한 자바 구현

- 여러 회사에서 자바 표준 스펙에 맞추어 실제 작동하는 자바 프로그램을 개발한다
- 각각 장단점이 있다. 예를 들어 **Amazon Corretto**는 AWS에 최적화되어 있다
- 각 회사들은 대부분 윈도우, MAC, 리눅스 같이 다양한 OS에서 작동하는 버전의 자바도 함께 제공한다

스펙을 보고 여러 회사에서 구현하는 것들:

- 자바 컴파일러
- 자바 실행 라이브러리
- 자바 가상 머신(JVM)

## 자바 컴파일과 실행

자바 프로그램은 **컴파일**과 **실행** 두 단계를 거친다.

```text
┌──────────┐     javac       ┌──────────┐      java       ┌──────────┐
│          │   (컴파일러)      │          │    (프로그램)     │          │
│ Hello.   │ ──────────────▶ │ Hello.   │ ──────────────▶ │   JVM    │
│ java     │                 │ class    │                 │  실행     │
│          │                 │          │                 │          │
└──────────┘                 └──────────┘                 └──────────┘
  소스코드                    바이트코드                   자바 가상 머신
```

### 컴파일 단계

1. `Hello.java` 같은 자바 소스 코드를 개발자가 작성한다
2. 자바 컴파일러(`javac`)를 사용해서 소스 코드를 컴파일한다
3. `.java` 파일이 `.class` 파일로 변환된다
4. 자바 소스 코드를 **바이트코드**로 변환하며, JVM에서 더 빠르게 실행될 수 있게 최적화하고 문법 오류도 검출한다

### 실행 단계

1. 자바가 제공하는 `java` 프로그램을 사용한다
2. **자바 가상 머신(JVM)**이 실행되면서 프로그램이 작동한다

## IDE와 자바

### IntelliJ를 통한 자바 설치 관리

```text
┌─────────────────────────┐
│    IDE - IntelliJ       │──────▶  OpenJDK Java
│    자바 설치 관리          │──────▶  Adoptium Java
│                         │──────▶  Amazon Corretto
└─────────────────────────┘
              │
┌─────────────────────────┐
│         OS              │
└─────────────────────────┘
```

인텔리제이는 내부에 자바를 편리하게 설치하고 관리할 수 있는 기능을 제공한다. 이 기능을 사용하면 인텔리제이를 통해 자바를 편리하게 다운로드 받고 실행할 수 있다.

### IntelliJ를 통한 컴파일, 실행 과정

```text
┌──────────┐     javac       ┌──────────┐      java       ┌──────────┐
│ Hello.   │ ──────────────▶ │ Hello.   │ ──────────────▶ │   JVM    │
│ java     │                 │ class    │                 │  실행     │
└──────────┘                 └──────────┘                 └──────────┘
       ▲                           ▲
       │                           │
┌──────────────┐          ┌──────────────┐
│  IntelliJ    │          │  IntelliJ    │
│  자바 설치     │          │  자바 설치     │
│  관리         │          │  관리         │
└──────────────┘          └──────────────┘
```

- **컴파일**: 자바 코드를 컴파일하려면 `javac`를 직접 사용해야 하는데, 인텔리제이는 자바 코드를 실행할 때 이 과정을 **자동으로 처리**해준다
- 프로젝트의 `out` 폴더에 가보면 컴파일된 `.class` 파일이 있는 것을 확인할 수 있다
- **실행**: 자바를 실행하려면 `java` 프로그램에 컴파일된 `.class` 파일을 지정해주면 된다 (확장자는 제외)
- 인텔리제이에서 자바 코드를 실행하면 **컴파일과 실행을 모두 한 번에** 처리한다

## 자바와 운영체제 독립성

### 일반적인 프로그램

```text
            ┌──────────────────┐
            │ 윈도우 프로그램      │
            │    .exe          │
            └──────────────────┘
            ╱        │        ╲
           ▼         ▼         ▼
     ┌─────────┐ ┌─────────┐ ┌─────────┐
     │ 윈도우    │ │  MAC    │ │ 리눅스   │
     │   OS    │ │   OS    │ │   OS    │
     └─────────┘ └─────────┘ └─────────┘
        ✅           ❌           ❌
```

- 일반적인 프로그램은 다른 운영체제에서 실행할 수 없다
- 윈도우 프로그램은 MAC이나 리눅스에서 작동하지 않는다
- 윈도우 프로그램은 윈도우 OS가 사용하는 명령어들로 구성되어 있기 때문에 다른 OS와는 호환되지 않는다

### 자바 프로그램

```text
     ┌──────────┐     javac       ┌──────────┐
     │ Hello.   │ ──────────────▶ │ Hello.   │
     │ java     │                 │ class    │
     └──────────┘                 └──────────┘
                                  ╱    │    ╲
                                 ▼     ▼     ▼
                          ┌────────┐┌────────┐┌────────┐
                          │ 윈도우   ││  MAC   ││ 리눅스  │
                          │ 자바    ││  자바   ││  자바   │
                          └────────┘└────────┘└────────┘
                              │         │         │
                          ┌────────┐┌────────┐┌────────┐
                          │ 윈도우   ││  MAC   ││ 리눅스  │
                          │  OS    ││   OS   ││   OS   │
                          └────────┘└────────┘└────────┘
                              ✅        ✅        ✅
```

- 자바 프로그램은 **자바가 설치된 모든 OS에서 실행**할 수 있다
- 자바 개발자는 특정 OS에 맞추어 개발하지 않아도 된다. **자바에 맞추어 개발하면 된다**
- OS 호환성 문제는 자바가 해결한다
- `Hello.class`와 같이 컴파일된 자바 파일은 모든 자바 환경에서 실행할 수 있다
- 각 OS용 자바는 해당 OS가 사용하는 명령어들로 구성되어 있다. 개발자는 각 OS에 맞도록 자바를 설치하기만 하면 된다

## 자바 개발과 운영 환경

```text
  개발자 PC                                    서버(AWS)
┌──────────────────────┐                    ┌──────────────────────┐
│                      │                    │                      │
│  Mac/Windows OS      │    서버에 배포        │  리눅스 OS             │
│  OpenJDK             │ ─────────────────▶ │  Amazon Corretto     │
│                      │                    │                      │
│  Hello.java          │                    │  Hello.class         │
│     ↓ javac          │                    │     ↓ java           │
│  Hello.class         │                    │  JVM 실행             │
│                      │                    │                      │
└──────────────────────┘                    └──────────────────────┘
```

- 개발할 때 사용하는 자바와 서버에서 실행할 때 사용하는 자바가 **다를 수 있다**
- 개발자들은 개발의 편의를 위해서 윈도우나 MAC OS를 주로 사용한다
- 서버는 주로 **리눅스**를 사용한다. 만약 AWS를 사용한다면 **Amazon Corretto** 자바를 리눅스 서버에 설치하면 된다
- 자바의 운영체제 독립성 덕분에 각각의 환경에 맞추어 자바를 설치하는 것이 가능하다

## 환경설정

- **IntelliJ** 사용을 권고한다
- IntelliJ 사용 시 기본 언어팩은 **영어**를 추천한다
- 프로젝트 생성 시:
  - vendor는 아무거나 상관없지만, Mac/Windows/CPU 칩 종류에 따라 알맞게 설정한다
  - location 설정은 건드리지 않는다
  - Advanced Settings도 손대지 않는다

## 자바 기본 문법

### 대소문자 구분

자바 언어는 **대소문자를 구분**한다. 대소문자가 다르면 오류가 발생할 수 있다.

### 파일명과 클래스명

**파일명과 클래스 이름이 같아야 한다.**

```java
// HelloJava.java

public class HelloJava {
    public static void main(String[] args) {
        System.out.println("hello java");
    }
}
```

- 자바는 `main(String[] args)` 메서드를 찾아서 프로그램을 시작한다 (프로그램의 시작점)

### 유용한 단축어

- `psvm` → `public static void main(String[] args) {}`
- `sout` → `System.out.println();`
