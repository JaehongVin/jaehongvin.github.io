## 스택 (Stack)

스택은 **나중에 넣은 것이 먼저 나오는(LIFO, Last In First Out)** 자료구조다.

```text
push →  [ 10 ]  ← pop
        [ 20 ]
        [ 30 ]  ← bottom
```

접시를 쌓는 것과 같다. 위에서만 넣고 뺄 수 있다.

### 시간 복잡도

| 연산 | 복잡도 | 이유 |
|------|--------|------|
| push (삽입) | O(1) | 맨 위에 바로 추가 |
| pop (삭제) | O(1) | 맨 위에서 바로 제거 |
| peek (최상단 확인) | O(1) | 인덱스 접근 |
| 탐색 | O(n) | 처음부터 순서대로 확인 |

### JS 구현

JS 배열의 `push`·`pop`이 스택 연산과 정확히 일치한다.

```javascript
const stack = [];

stack.push(10);  // [10]
stack.push(20);  // [10, 20]
stack.push(30);  // [10, 20, 30]

stack.pop();     // 30 반환, [10, 20]
stack.at(-1);    // peek: 20 (제거하지 않고 확인)
```

### FE에서의 스택

스택은 브라우저와 JS 엔진 내부에서 직접 쓰인다.

**JS 콜 스택 (Call Stack)**

함수를 호출하면 실행 컨텍스트가 스택에 쌓이고, 함수가 끝나면 제거된다.

```javascript
function c() { /* 실행 */ }
function b() { c(); }
function a() { b(); }

a();

// 콜 스택 변화:
// [a] → [a, b] → [a, b, c] → [a, b] → [a] → []
```

스택 오버플로(Maximum call stack size exceeded)는 재귀가 종료 조건 없이 계속 쌓일 때 발생한다.

**브라우저 뒤로가기**

방문한 페이지를 스택에 쌓고, 뒤로가기 시 `pop`한다.

**Undo / Redo**

편집 내역을 스택에 쌓고, 실행취소 시 `pop`한다. Redo는 별도의 스택을 사용한다.

## 큐 (Queue)

큐는 **먼저 넣은 것이 먼저 나오는(FIFO, First In First Out)** 자료구조다.

```text
enqueue →  [ 30 | 20 | 10 ]  → dequeue
             rear        front
```

줄을 서는 것과 같다. 뒤에서 들어오고 앞에서 나간다.

### 시간 복잡도

| 연산 | 복잡도 | 이유 |
|------|--------|------|
| enqueue (삽입) | O(1) | 맨 뒤에 바로 추가 |
| dequeue (삭제) | O(1) | 맨 앞에서 바로 제거 |
| peek (최전방 확인) | O(1) | 인덱스 접근 |
| 탐색 | O(n) | 처음부터 순서대로 확인 |

### JS 구현

JS 배열에서 `push`로 삽입하고 `shift`로 제거하면 큐처럼 쓸 수 있다. 하지만 `shift()`는 맨 앞 요소를 제거한 뒤 나머지를 전부 한 칸씩 당기는 작업이 필요해서 **O(n)**이다.

```javascript
// ❌ shift()는 O(n)
const queue = [];
queue.push(10);    // enqueue
queue.push(20);
queue.shift();     // dequeue: O(n) — 모든 요소를 앞으로 당김
```

요소 수가 많아지면 성능 문제가 생긴다. **인덱스 포인터**를 사용하면 O(1)로 구현할 수 있다.

```javascript
// ✅ 인덱스 포인터로 O(1) dequeue
const queue = [];
let head = 0;

queue.push(10);    // enqueue
queue.push(20);
queue.push(30);

queue[head++];     // dequeue: 10 반환, head = 1
queue[head++];     // dequeue: 20 반환, head = 2

// 실제 남은 요소: queue.slice(head)
```

실제 요소는 배열에서 제거하지 않고 `head` 포인터만 이동시킨다. 메모리 낭비가 우려된다면 일정 주기로 `queue.splice(0, head); head = 0;` 으로 정리할 수 있다.

### 변형

**덱 (Deque, Double-ended Queue)**

양쪽 끝에서 삽입·삭제가 모두 가능한 큐다. 스택과 큐의 특성을 동시에 가진다.

```javascript
const deque = [];
deque.push(10);      // 뒤에 삽입
deque.unshift(20);   // 앞에 삽입
deque.pop();         // 뒤에서 제거
deque.shift();       // 앞에서 제거
```

**우선순위 큐 (Priority Queue)**

순서가 아닌 우선순위(가중치)에 따라 높은 것부터 꺼낸다. 내부적으로 힙(Heap)으로 구현한다. JS에는 내장 우선순위 큐가 없어서 직접 구현해야 한다.

### FE에서의 큐

**이벤트 루프 (Event Loop)**

JS는 싱글 스레드지만 비동기 처리가 가능한 이유가 큐 구조에 있다.

```text
Call Stack           Task Queue            Microtask Queue
──────────           ──────────            ───────────────
[현재 실행 중]   ←   [setTimeout CB]   ←   [Promise .then]
                     [click handler]        [queueMicrotask]
```

- **태스크 큐 (Macrotask Queue)**: `setTimeout`, `setInterval`, I/O 콜백이 대기
- **마이크로태스크 큐 (Microtask Queue)**: `Promise.then`, `queueMicrotask`, `MutationObserver`가 대기

콜 스택이 비면 마이크로태스크 큐를 먼저 전부 소진한 뒤, 태스크 큐에서 하나씩 꺼낸다. 마이크로태스크가 태스크보다 우선순위가 높은 이유다.

```javascript
console.log('1');

setTimeout(() => console.log('2'), 0);  // 태스크 큐

Promise.resolve().then(() => console.log('3'));  // 마이크로태스크 큐

console.log('4');

// 출력 순서: 1 → 4 → 3 → 2
```

## 스택 vs 큐

| | 스택 | 큐 |
|---|---|---|
| 원칙 | LIFO (후입선출) | FIFO (선입선출) |
| 삽입 | push: 맨 위 | enqueue: 맨 뒤 |
| 삭제 | pop: 맨 위 | dequeue: 맨 앞 |
| FE 예시 | 콜 스택, undo, 뒤로가기 | 태스크 큐, 마이크로태스크 큐 |
| JS 구현 | `push` / `pop` | `push` / 인덱스 포인터 |

---

## 정리

- 스택은 LIFO — 나중에 넣은 것이 먼저 나온다
- 큐는 FIFO — 먼저 넣은 것이 먼저 나온다
- JS에서 스택은 `push/pop`으로 그대로 구현, 큐는 `shift()` 대신 인덱스 포인터를 써야 O(1)
- JS 콜 스택은 스택, 이벤트 루프의 태스크·마이크로태스크는 큐로 동작한다
- 마이크로태스크 큐는 태스크 큐보다 먼저 소진된다
